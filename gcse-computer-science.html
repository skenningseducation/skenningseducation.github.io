GCSE/iGCSE Computer Science

A structured, exam-focused course covering the theory and programming needed for GCSE/iGCSE success. Suitable as supplementary support alongside school or as a main provision pathway.

Maximum class size: 10 Theory + Python programming Exam technique throughout
Day 1 resources included: full lesson notes (Word/PDF), exam-style questions with worked answers, and access to past papers for the student’s exam board.

What we cover

Area Focus
Computer systems CPU, memory (RAM/ROM), storage, input/output, operating systems, and how programs run.
Data representation Binary, hexadecimal, characters, images, sound, file sizes, and understanding overflow/rounding where relevant.
Algorithms Trace tables, pseudocode, flowcharts, searching/sorting basics, decomposition and logical thinking.
Programming (Python) Writing correct code, debugging, test data, and producing exam-ready solutions under timed conditions.
Networks & security Internet basics, protocols, cybersecurity principles, threats, and practical defensive thinking.
Logic & databases Boolean logic, truth tables, logic gates, and database fundamentals (as required by board).

How lessons run

Weekly live teaching

  • Teacher-led explanation and worked examples
  • Guided practice during the lesson
  • Frequent questions and checks for understanding
  • Clear routines so students build confidence quickly

Programming practice

  • Step-by-step Python skill building
  • Common exam patterns and “what they’re really asking”
  • Debugging methods students can reuse
  • Timed mini-tasks to build speed and accuracy

36-week outline

A practical week-by-week plan adapted to the student’s exam board and class pace.

Weeks Focus Typical outcomes
1–6 Foundations: systems, CPU/memory, binary & basics Strong core understanding and confidence with the fundamentals.
7–12 Data representation + algorithms Reliable exam methods for conversions, file sizes, tracing and pseudocode.
13–18 Python fundamentals Students can write correct code using sequence/selection/iteration and test it properly.
19–24 Networks, security, logic and databases Students can answer typical theory questions confidently and accurately.
25–30 Programming depth + exam patterns Students become quicker and more consistent on programming questions.
31–36 Revision + past papers Timed practice, mark-scheme technique, and improved performance under pressure.
